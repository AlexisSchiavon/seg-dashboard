<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /reports/{id}/download | Untrusted report_id; response streams a file from disk |
| DB row → filesystem path | report.file_path used to read a file off disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-unauth-dl | Elevation of Privilege | GET /reports/{id}/download | mitigate | Router-level `dependencies=[Depends(get_current_user)]` (set in 05-02) covers the download endpoint; `test_download_requires_auth` asserts 401 for unauthenticated requests. Severity: HIGH. |
| T-stale-path | Information Disclosure | report.file_path on disk | mitigate | `os.path.exists` check returns 404 instead of leaking a stack trace / 500 when the DB row points at a missing file (Pitfall 4). file_path is built from numeric talent.id slug + validated month, so no traversal. Severity: MEDIUM. |
| T-idor-download | Elevation of Privilege | report_id direct object reference | accept | All authenticated users share one access level (RBAC deferred per FUT-03 / Out of Scope). Any logged-in user may download any report by design for this internal single-tenant tool. Severity: LOW. |
</threat_model>
