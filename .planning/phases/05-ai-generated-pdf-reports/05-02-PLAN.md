<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /reports/generate | Untrusted talent_id + month cross into file-path construction and DB query |
| service → Claude API | Deal titles/talent name leave the system inside the JSON payload |
| service → filesystem | PDF written to reports/{slug}/ — slug must not enable path escape |
| service → WeasyPrint | base_url could be an SSRF vector if user-controlled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-path-traversal | Tampering | `reports/{slug}/{month}.pdf` path build | mitigate | `_slug` returns `str(talent.id)` (numeric only, no separators); `month` validated by ReportGenerate pattern `^\d{4}-\d{2}$` (05-01). No user string reaches the path raw. Severity: HIGH. |
| T-ssrf | Information Disclosure | WeasyPrint `base_url` | mitigate | `_render_pdf` passes the literal fixed string `"."` as base_url; never a user value. Severity: HIGH. |
| T-prompt-injection | Tampering | Deal titles inside Claude payload | accept | Deal data is embedded as structured JSON in the user message; SYSTEM_PROMPT instructs Claude to treat it as data only and never invent figures. Numbers in the PDF come from Python regardless of Claude output, so injected text cannot alter reported figures. Severity: LOW. |
| T-claude-numbers | Tampering | Claude narrative substituting figures | mitigate | All numeric values in the PDF appendix and payload are computed by kpis/funnel/leads services in Python; Claude output is used ONLY for the 3 prose sections, never parsed for numbers. Enforces STATE.md hard rule. Severity: HIGH. |
</threat_model>
