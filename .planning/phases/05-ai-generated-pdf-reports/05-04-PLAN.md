<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Claude narrative → DOM (innerHTML) | Model-generated text reflected into the preview card |
| browser → /reports/* | Auth-cookie-bearing requests to backend endpoints |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-xss | Spoofing / Tampering | #pdf-body / #report-history innerHTML | mitigate | All Claude narrative strings (resumen_ejecutivo, deals_destacados, recomendacion) and all API strings pass through `escHtml()` before innerHTML insertion (Pitfall 6). Verified by grep gate + checkpoint. Severity: HIGH. |
| T-unauth-ui | Elevation of Privilege | /reports/* fetches | mitigate | reports.js uses `apiFetch`, which carries the JWT cookie and redirects to /login.html on 401; backend endpoints are auth-protected (05-02/05-03). Severity: MEDIUM. |
| T-csrf-gen | Tampering | POST /reports/generate from browser | accept | Same-origin Vanilla JS app with cookie auth; no cross-origin form posts in scope; generation is read-only against external systems (only writes local SQLite + disk). Severity: LOW. |
</threat_model>
