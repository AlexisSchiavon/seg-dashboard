<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /agent/chat | Untrusted user message + history JSON crosses into the backend |
| backend → Anthropic API | Prompt (system + history + message) sent to external LLM |
| Anthropic API → backend | tool_use requests and final prose returned (untrusted text) |
| _execute_tool → service layer | Tool inputs (talent_id, limit) from LLM cross into DB queries |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-6-01 | Tampering | user message → tool execution (prompt injection) | mitigate | System prompt hard rules (D-78/D-79) instruct refusal of write/override requests; ALL 11 tools are read-only wrappers — no write function exists to call even if injection succeeds; tool results stay inside tool_result blocks (never re-injected into system/user text — RESEARCH Section 5.1) |
| T-6-03 | Denial of Service | /agent/chat → high-cost LLM calls | mitigate | MAX_TOOL_CALLS = 5 hard ceiling (D-77) caps Claude round-trips per request; ChatRequest.message max_length=2000 and history max_length=20 cap input size; (server rate-limiting deferred to v2 per RESEARCH 5.3 — single admin user, low risk) |
| T-6-04 | Information Disclosure | conversation history across sessions | mitigate | History is client-supplied per request and never persisted server-side (D-73 stateless); ChatRequest.history capped at 20; no cross-session storage exists |
| T-6-05 | Elevation of Privilege | unauthenticated agent access | mitigate | Router-level `dependencies=[Depends(get_current_user)]` protects POST /agent/chat (returns 401); verified by test_chat_requires_auth |
| T-6-06 | Tampering | LLM-supplied talent_id/limit → DB query | accept | int() coercion in _execute_tool; service functions are read-only and raise ValueError for unknown IDs (caught → 502); no SQL injection surface (ORM parameterized) |
</threat_model>
