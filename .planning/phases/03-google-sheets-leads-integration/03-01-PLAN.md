| Boundary | Description |
|----------|-------------|
| Google Sheets → app | Untrusted external content (Asunto, Remitente_Nombre, Status) enters via gspread |
| Browser → /leads API | Authenticated client requests cross into the app |
| Service account JSON → google-auth | Credential material loaded from env into the gspread client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | sync_sheets error handling | mitigate | On exception persist only `str(exc)` to SyncLog.error_message — never log gspread client/worksheet/response objects (may reflect SA credentials) |
| T-03-02 | Tampering | app/integrations/sheets.py | mitigate | Read-only client: only ws.get_all_values()/open_by_key/worksheet are called; no update/append_row/insert/delete anywhere (CLAUDE.md critical constraint) |
| T-03-03 | Elevation of Privilege | /leads router | mitigate | Router-level `dependencies=[Depends(get_current_user)]` — GET /leads/summary returns 401 without a valid JWT cookie |
| T-03-04 | Tampering / Input Validation | SheetLeadRow | mitigate | Pydantic model with field_validators coerces all Sheet-sourced values (score, bools) before DB write; bad data → None/False, never a crash |
| T-03-SC | Tampering | gspread/google-auth installs | accept | No new packages installed this phase; both pre-verified in STACK.md Package Legitimacy Audit (Approved) |
