<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser cookie -> /talents/* | All catalog mutations require a verified auth cookie |
| Request body -> Talent/TalentProduct | Untrusted JSON validated by Pydantic before persistence |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | /talents/* routes | mitigate | Router-level `dependencies=[Depends(get_current_user)]` guards every route; test_talents_require_auth asserts 401 unauthenticated |
| T-02-02 | Tampering / Input Validation | POST/PATCH talent bodies | mitigate | Pydantic v2 schemas (TalentCreate/TalentUpdate) auto-validate; FastAPI returns 422 on invalid input |
| T-02-03 | Tampering | talent/product DB access | mitigate | SQLAlchemy ORM parameterized queries (`db.get`, `filter`); no raw SQL |
| T-02-04 | Information Disclosure | route responses | mitigate | `response_model=` Pydantic schemas only (Anti-Pattern 4) — no raw ORM objects / internal columns leaked |
| T-02-05 | Tampering | product delete cross-talent | mitigate | DELETE verifies `product.talent_id == talent_id` before deleting (404 on mismatch) |
</threat_model>
