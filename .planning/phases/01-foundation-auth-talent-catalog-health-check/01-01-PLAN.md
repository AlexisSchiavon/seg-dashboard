<objective>
Stand up the Walking Skeleton for the SEG Talent Intelligence Dashboard: the thinnest end-to-end slice that proves the entire architecture works. A user opens the login page, submits the seeded admin credentials, the server verifies them with pwdlib[argon2], issues a PyJWT HS256 token, sets it as an httpOnly cookie, and a protected probe endpoint then accepts the cookie and rejects requests without it. This establishes the FastAPI app factory, pydantic-settings config, SQLAlchemy 2.0 + SQLite (WAL) database layer, Alembic migrations, the `auth/` package, the dark-theme login UI, and the pytest test infrastructure (Wave 0) that Plans 02 and 03 build on.

Purpose: Every later module (talent CRUD, health, Pipedrive, Sheets, Trello, AI) depends on this auth + DB + session foundation. Getting the cookie/secret/WAL/hashing decisions right here prevents expensive retrofits (PITFALLS 1-4, 6).
Output: A runnable, login-protected FastAPI app on Python 3.12 with an initial Alembic migration, an idempotent admin seed, and green auth/database/security tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> /auth/login | Untrusted credentials cross into the API; form data parsed by OAuth2PasswordRequestForm |
| Browser cookie -> protected routes | JWT in httpOnly cookie is decoded and trusted only after HS256 verification |
| .env -> app config | SECRET_KEY / ADMIN_PASSWORD secrets enter the process; must never reach git or logs |
| uv add -> dependency tree | Third-party packages enter the runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | /auth/login error response | mitigate | Generic "Invalid email or password" for both unknown-email and wrong-password (D-06), asserted by test_login_invalid_credentials |
| T-01-02 | Tampering | get_current_user JWT decode | mitigate | `jwt.decode(..., algorithms=["HS256"])` explicit allowlist prevents alg=none / algorithm-confusion; reject on InvalidTokenError |
| T-01-03 | Information Disclosure / Session | login Set-Cookie | mitigate | httpOnly + SameSite=Lax + Secure driven by settings.COOKIE_SECURE (env, not hardcoded) so prod gets Secure without breaking local http dev (Pitfall 4) |
