## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| PIPE-05 is NOT 6 stages from Pipedrive | Pipedrive pipeline 2 has only 4 stages (id 6 Llamada, 7 Cotizacion, 8 Negociacion, 9 Contrato y factura). "En ejecucion" and "Cobranza" do not exist in Pipedrive at all - they are sourced from Trello (Phase 4), confirmed live and by the user. | blocking | Wave 2 (02-02) `funnel.py` MUST NOT call `/stages` expecting 6 results. Design the funnel as: 4 real counts from Pipedrive `Deal.stage_id` (6-9) + 2 placeholder/empty buckets for "En ejecucion"/"Cobranza" until Phase 4 wires Trello in. See comment in `app/sync/jobs.py` near `stage_name_by_id`. |
| Pipedrive v2 field shape != assumed v1 shape | `/dealFields` v2 returns `field_name`/`field_code`, not `name`/`key`. The original build_field_maps + test mocks used the wrong (v1-ish) shape, which passed tests but raised `KeyError: 'name'` against the real API and crashed every sync (SyncLog status=error). Now fixed (`app/integrations/pipedrive.py` + `tests/conftest.py`). | advisory | Before writing mocks for any new Pipedrive/Trello/Sheets endpoint, make one live read-only call and inspect actual response keys - do not trust RESEARCH.md-derived shapes blindly. |
| Live seg.db can drift from alembic head | The live `seg.db` was 1 revision behind (missing `deals`/`deal_stage_events`/`sync_logs` tables), causing `OperationalError: no such table: sync_logs` on first live sync attempt. Fixed via `alembic upgrade head`. | advisory | Run `alembic current` vs `alembic heads` before any live-verify checkpoint that touches new tables. |
| .env admin creds can drift from seeded user | `.env` had `ADMIN_EMAIL=santillan@talentagency.mx` but the only seeded user was `admin@santillanent.com` from earlier Phase 1 setup - login with `.env` creds failed with "Invalid email or password". Fixed by running `seed_admin` (idempotent/additive). | advisory | If a live-verify login fails, check the `users` table vs `.env` ADMIN_EMAIL before assuming a code bug. |
