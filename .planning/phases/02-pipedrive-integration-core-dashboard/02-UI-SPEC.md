## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, dot/avatar offsets |
| sm | 8px | Compact element spacing (kpi-grid gap, donut-wrap gap, bar-chart gaps) |
| md | 16px | Default element spacing (section padding, talent-selector padding, donut-wrap gap) |
| lg | 24px | not directly used by mockup classes but reserved for larger section breaks if introduced |
| xl | 32px | Layout gaps between major dashboard sections (if needed beyond `.spacer`) |
| 2xl | 48px | Major section breaks (rare; mockup uses `.spacer` at 16px between page sections instead) |
| 3xl | 64px | Page-level spacing — not used in mockup; reserve for future full-page empty states |

Exceptions:
- `.card` padding is `18px` (not a multiple of 4) — this is an EXISTING Phase 1 token in `frontend/css/styles.css` (`.card { padding: 18px; }`). Do not change it; keep using `.card` as-is for consistency with the login page and Phase 1 work. New Phase 2 cards should reuse `.card` rather than introduce a new padding value.
- `.kpi` padding is `14px 16px` (14px not a multiple of 4) — EXISTING mockup value, carry forward as-is for visual consistency with the established KPI grid pattern.
- `.act-icon`/`.source-icon` are `30px`/`32px` square (icon-badge touch targets) — EXISTING mockup values, not part of the 8-point scale but acceptable as fixed icon-badge sizes; do not introduce new sizes for new icon badges in this phase (reuse `.act-icon` at 30px for activity feed, donut legend swatches at 8-10px per existing inline style pattern).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`body` default, DM Sans) | 400 (regular) | 1.5 |
| Label | 11px (`.kpi-label`, `.section-title`, `.act-time`, `.deal-tipo`, `.source-count`) | 500 (medium) | 1.5 |
| Heading | 13px (`.deal-brand`, `.rank-name`, `.act-main`, `.source-name`, `.tc-name`) | 500 (medium) | 1.2 |
| Display | 26px (`.kpi-val`, DM Mono) | 600 (semibold) | 1.2 |

Notes:
- This is the EXISTING type scale from `.planning/reference/mockup.html` — Phase 2 introduces no new font sizes or weights. The 4-size/2-weight constraint is satisfied: sizes {11, 13, 14, 26}, weights {400 regular for body/labels-when-not-bold, 500/600 used selectively as established}. Treat 500 and 600 as the "semibold family" — 600 reserved for `.kpi-val` and `.login-title`/headings only; 500 used for labels, names, and `.btn` text.
- `.f-n` and `.deal-amt`/`.rank-num` use DM Mono at 12-13px for numeric alignment — reuse this pattern for any new numeric values introduced in Phase 2 (commission amounts, funnel counts, donut percentages).
- `.section-title` uses `text-transform: uppercase; letter-spacing: 0.8px` — reuse verbatim for any new section headers (e.g., "Categorías de marca", "Oportunidades perdidas", "Cuello de botella").

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` `#0c0c0e` | Page background (`html`, `body`) |
| Secondary (30%) | `--bg3` `#18181c` (cards), `--bg2` `#111114` (nav/tabbar), `--bg4`/`--bg5` (form inputs, track fills) | `.card`, `.kpi`, `.talent-card`, `.nav`, `.tabbar`, `.f-track`, `.donut` track segments |
| Accent (10%) | `--accent` `#e8520a` (orange) | RESERVED FOR: primary CTA buttons (`.btn.primary` — "Sincronizar ahora"), the "Pipeline total" KPI value (`.kpi-val.accent`), active tab indicator state if added, login title accent span, and the funnel/donut segment representing the talent's OWN data series when highlighted. Do not use accent for generic links, icons, or decorative elements. |
| Destructive | `--red` `#c43232` / `--redT` `#f07070` (text-on-dark variant) / `--redD` `rgba(196,50,50,0.12)` (background tint) | Reserved for: error banners/messages (`#error-message` pattern from login), sync-failure warning banner text accents if escalated to error severity, and any future "eliminar"/destructive action confirmation — none exist as user-facing destructive actions in Phase 2 (read-only dashboard + sync trigger). |

Additional established semantic colors (categorical, NOT part of the 60/30/10 split — pre-existing in mockup, reused for KPI/data categorization):
- `--green` `#1a9e6e` / `--greenT` `#3dcf96` / `--greenD` — positive/success state (e.g., "Calificados" KPI, won deals, success activity icon)
- `--amber` `#c97c14` / `--amberT` `#f0a93a` / `--amberD` — warning state (`.alert.warn`, "En negociación" KPI, bottleneck alert)
- `--blue` `#2472c8` / `--blueT` `#6aabf0` / `--blueD` — informational state (`.alert.info`, "Leads totales" KPI — N/A this phase, lead-source activity icon)
- `--purple` `#6b54d6` / `--purpleT` `#a594f0` / `--purpleD` — categorical accent for talent ranking avatars and "Cerrados" KPI

Accent reserved for: "Sincronizar ahora" button (`.btn.primary`), the "Pipeline total" KPI value, and the active/selected talent's highlighted series in funnel bars and the brand-category donut when a specific talent context is shown. Never apply accent to body text, generic icons, borders, or more than one KPI tile per screen.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | **"Sincronizar ahora"** (Resumen tab, per D-22) — button uses `.btn.primary` (accent color). Async behavior per D-23: label changes to **"Sincronizando..."** while in-flight (disable re-click via Pitfall 5 concurrency guard — if a sync is already running, clicking shows a toast **"Ya hay una sincronización en curso"** instead of starting a second one). |
| Sync success toast | **"Sync completado — {N} deals actualizados"** (D-23; `{N}` = `SyncLog.records_synced`) |
| Sync failure banner | **"No se pudo sincronizar — mostrando datos de hace {X} horas"** (D-24; `.alert.warn` style, `{X}` computed from last successful `SyncLog.finished_at`). Banner appears on all tabs (persistent, per D-24), positioned at the top of the page content below the tabbar. |
| Last-sync indicator | **"Última sync: hace {X} min"** (D-21; replaces/augments the `.live-pill` "En vivo" indicator in `.nav-right`, using the existing `.live-pill`/`.live-dot` markup — green dot = synced within last sync interval, amber dot = sync failed/stale per D-24). |
| "Sin cotizar" label | **"Sin cotizar"** — `.pill` badge style, neutral color (use `--text2`/`--bg5`, not a semantic color), applied to deals where `Deal.value == 0` (PIPE-03). |
| "Sin talento asignado" bucket | **"Sin talento asignado"** — appears as the LAST row in the talent ranking (`.rank-row` pattern, but without a rank medal — use a plain numeral or dash `—` in place of `.rank-num gold/silver/bronze`) and is included in global Pipeline/KPI totals per D-17. Avatar uses neutral gray background (`--bg5`) with `?` glyph instead of initials. |
| Empty state heading (no deals synced yet) | **"Aún no hay datos de Pipedrive"** |
| Empty state body | **"Ejecuta la primera sincronización para ver el funnel, el ranking de talentos y las KPIs. Usa el botón 'Sincronizar ahora' arriba."** — shown in place of KPI grid / ranking / funnel cards before the first successful `SyncLog` exists. Reuses `.card` with centered text, no special new component needed. |
| Empty state — lost opportunities (no lost deals for talent) | **"Sin oportunidades perdidas este periodo"** — shown inside the `.card` that would otherwise list `.deal-row` lost-opportunity rows (Por talento, D-25). |
| Empty state — brand category donut (no closed/won deals with brand category) | **"Sin categorías de marca registradas todavía"** — shown in place of `.donut-wrap` (Por talento, D-26). |
| Bottleneck alert (insufficient data) | **"Datos insuficientes para detectar cuellos de botella"** — `.alert.info` (not `.alert.warn`, since this is informational, not a problem) — shown when total deal sample size is below the minimum threshold (RESEARCH.md Pattern 4). |
| Bottleneck alert (detected) | **"Cuello de botella detectado: solo el {X}% de los deals en {EtapaA} avanzan a {EtapaB}."** — `.alert.warn`, replacing the mockup's hardcoded "41% vs 60% industria" copy (no external industry benchmark exists per RESEARCH.md A1 — do not reference "industria" or "promedio de la industria" anywhere in the UI). |
| Stuck-deals alert (secondary, optional per RESEARCH.md Pattern 4) | **"{N} deals llevan más de 14 días sin avanzar de etapa."** — `.alert.info`, paired below the bottleneck `.alert.warn` if both are shown. |
| Lost opportunities — per-reason summary (D-25) | Format: **"{N} por {razón}"** for each of the 5 razones (e.g., "3 por presupuesto insuficiente, 2 por timing") — rendered as a comma-separated line or small `.pill` row above the `.deal-row` list, using the exact 5 razón labels from PROJECT.md: Presupuesto insuficiente, No respondieron, Eligieron otro talento, Campaña cancelada, Sin fit estratégico. |
| Lost opportunity row pill | Each `.deal-row` in the lost-opportunities list shows a `.pill` with the resolved razón de pérdida label (never a raw integer — see RESEARCH.md Pitfall 2). |
| Brand category donut legend (D-26/D-27) | Each `.donut-legend` row shows: brand category label (one of the 6 from PROJECT.md: Moda/Retail, Alimentos/Restaurantes, Agencias, Medios y Entretenimiento, Educación/Gobierno, Otros) + percentage of deals (by COUNT, per D-27, not revenue) + deal count, e.g., **"Moda/Retail — 35% (7 deals)"**. |
| Resumen — sections without Phase 2 data source | "Leads totales", "Calificados", and "Insights IA" KPIs/sections (mockup) have NO data source until Phase 3 (Sheets) / Phase 5-6 (AI). **Decision: hide these sections entirely from Phase 2's Resumen tab** rather than show "Próximamente" placeholders — ROADMAP Phase 2 promises only "global KPIs, talent ranking, and activity feed from real Pipedrive data" (DASH-01), and empty placeholder sections would imply incomplete/broken functionality. Resumen tab for Phase 2 shows: KPI grid (Pipeline total, En negociación, Cerrados, En campaña — all Pipedrive-derived; drop "Leads totales"/"Calificados" tiles), Ranking de talentos, Volumen de deals por talento (bar chart), Actividad reciente. The "Insights IA" section is omitted entirely (re-introduced in Phase 5/6). |
| Por talento — "Fuente de leads" section | REMOVED per D-28 — replaced by "Categorías de marca" (donut) and "Oportunidades perdidas", in the order: KPIs → Funnel → Deals activos → Categorías de marca → Oportunidades perdidas. |
| Destructive actions | None in Phase 2 — this phase is read-only dashboard + a sync trigger (idempotent, non-destructive). No delete/remove UI elements are introduced. The talent-product mapping correction UI (D-19) reuses Phase 1's existing `/talents/{id}/products` CRUD, which is out of this phase's UI scope. |
