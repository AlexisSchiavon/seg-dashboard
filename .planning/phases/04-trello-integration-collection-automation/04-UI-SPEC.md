| Token | Value | Usage |
|-------|-------|-------|

| 2xl | 48px | Major section spacing |

Exceptions (Phase 4 design decisions only):

- Touch targets (`.talent-card`, `.tab`, `.btn`): minimum 44px tall ‚Äî iOS/Android minimum touch target per WCAG 2.5.5 / Apple HIG; accessibility standard overrides the 8-point grid constraint. Exempted as accessibility requirement.
- `.tl-node min-width: 80px` ‚Äî minimum width for timeline calendar nodes to accommodate readable date label + MXN amount on two lines within a horizontal-scroll layout. Exempted as component minimum, not a padding/gap token.

Source: all values extracted from `frontend/css/styles.css` ‚Äî pre-populated, no deviations introduced in Phase 4.

## Typography

Declared type scale: **4 sizes, 2 weight tiers**.

| Role | Size | Weight tier | Line Height | Font | Usage |
|------|------|-------------|-------------|------|-------|
| Label | 11px | Tier 1 ‚Äî 400 Regular | 1.4 | DM Sans / Sora (ALL CAPS) | `.kpi-label`, `.section-title`, `.medal-amount-label`, `.th-label`, `.tl-month`, form labels |
| Body | 14px | Tier 1 ‚Äî 400 Regular | 1.5 | DM Sans | Base `body`, `.alert-text`, `.ctable-name`, `.rank-name`, `.medal-name`, general prose |
| Heading | 14px | Tier 2 ‚Äî 600 Semibold | 1.4 | DM Sans | `.deal-brand`, `.ctable-amount`, `.th-name` sub-labels ‚Äî same size as Body, distinguished by weight only |
| Display | 24px | Tier 2 ‚Äî 600 Semibold | 1.1 | Sora / DM Mono | `.medal-num` (24px Sora), `.kpi-val` (DM Mono) |

**Weight tier notes:**
- Tier 1 (400 Regular): all labels and body copy.
- Tier 2 (600 Semibold): all headings and display numerics. The `.th-name` element (talent header) renders at 17px as a within-tier variant of the Heading role ‚Äî not a separately declared size. The `.kpi-val` element uses 500 Medium as a within-tier softening of Semibold for numeric readability; this is a single implementation detail under Tier 2, not a third weight tier.

**Primary visual anchor (Por talento tab):** The three KPI cards at the top of the Por talento tab (`.kpi-val` 24px DM Mono, Tier 2 Semibold) are the primary entry point for the eye ‚Äî the KPI cluster top-right draws attention first before the user scans the projection chart below.

Phase 4 additions must follow the same 4 roles ‚Äî no new font sizes or weight tiers introduced.

### Phase 4 Render Function Contracts

`renderIncomeProjection(data)` ‚Äî expects `Array<{month: string, cobrado: number, proyeccion: number, pendiente: number}>`. Month format: `"Jun 2026"`. All values in MXN (venta_total, not 70% commission per D-47).

`renderPaymentCalendar(data)` ‚Äî expects `Array<{month: string, amount: number}>`. Month format: `"Jun 2026"`.

`renderTopCampaigns(deals)` ‚Äî **Phase 4 change:** currently receives stage aggregates; must be updated to receive individual deal objects `Array<{title: string, amount: number, list_state: string}>` sorted by `amount` desc, top 3 sliced by backend or frontend.

`renderCampaignTable(deals, lostOpps)` ‚Äî **Phase 4 change:** first arg currently receives stage aggregates; must receive `Array<{title: string, amount: number, list_state: string, trello_card_id?: string}>` for active campaigns, plus the existing `lost_opportunities` array for the second argument.

## Interaction Contracts

### Income Projection Chart

- Stacked bars animate in on data load using existing `transition: height 0.8s cubic-bezier(0.4, 0, 0.2, 1)` on `.proj-bar-stack` children
- 4-month sliding window: 1 prior month + current month + 2 future months (D-49)
- Current month column has no visual distinction from others ‚Äî backend provides the data, frontend renders uniformly
- No click interaction on bars ‚Äî read-only display

### Payment Calendar Timeline

- Horizontal scroll on overflow (`.timeline` with `overflow-x: auto; scrollbar-width: none`)
- Each node shows: emoji icon `üìÖ`, month label (Sora 11px Tier 2 Semibold), amount in MXN green (`--greenT`)
- Connectors between nodes are `--bg5` horizontal lines
- No click interaction ‚Äî read-only display

### Top 3 Campaigns Medal Cards

- 3-column grid (`.medal-cards`) ‚Äî **Phase 4 change:** cards now show individual deal name and `list_state` badge instead of funnel stage aggregate
- Rank border colors: gold `rgba(240,169,58,0.28)`, silver `rgba(156,155,146,0.25)`, bronze `rgba(201,124,20,0.22)` ‚Äî existing
- Medal numbers: 1 = `#f0a93a`, 2 = `#9c9b92`, 3 = `#c97c14` ‚Äî existing
- No click interaction ‚Äî read-only

### Campaign Table

- **Phase 4 change:** rows now represent individual deals (title + list_state + amount) instead of stage aggregates
- Lost deals remain as final rows with `.sbadge.perdido` ‚Äî no change to this section
- Status dot (`.ctable-icon`) color maps to `list_state`: green for cobrado, teal for ejecucion, emerald for cobranza, red for perdido
- No click interaction ‚Äî read-only
